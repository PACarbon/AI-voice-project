# 开发板配置

因为开发板默认包含一个自启程序，并且开发板每次自动访问`/IOT`目录,这两件事都是我们不想要的，另外后续我们给开发板配置网络ip等等都需要使用，Linux 提供的 `类Vim`文本编辑工具，修改指定配置文件 `/etc/profile`

## 学习关键指令

1. 使用 `类Vim` 打开指定文件
```shell
# 6818开发板工具叫vi
vi 指定文件路径
# 例子上述配置文件
vi /etc/profile
```

2. vim 有两种模式 
    - 指令模式(默认)：该模式下只接受一些特殊指令如：保存、退出等无法编辑文本
    - 编辑模式：该模式下可以修改文本，但是无法保存和退出
```shell

指令模式切换编辑模式：按 i 键


编辑模式切换指令模式：按 Esc键
```

3. 退出的指令,在指令模式下输入 `:wq` 保存退出

4. 
```shell
# 修改我们的/etc/profile文件中 最后面两句前面加个#号将其注释
# cd /IOT
# ./iot
```

# 交叉编译

在一台电脑上，编译出能在另一台不同架构的设备上运行的程序。我们平时写代码、编译程序，大多是本地编译：
- 电脑：Windows /x86 架构
- 编译：在这台电脑上编译
- 运行：程序只能在这台电脑（或同架构X86电脑）上运行

而交叉编译是在 Windows x86 电脑 上编译生成能在 其他操作平台智能设备（ARM 架构） 运行的程序

## 为什么要用交叉编译？
最常见的 3 个原因：
1. 目标设备性能太弱
比如：路由器、单片机、树莓派、智能手表它们 CPU 慢、内存小，根本没法自己编译程序，只能靠电脑交叉编译后拷过去。
2. 开发效率高
电脑性能强，编译速度快几十倍。
3. 跨平台发布
一次写代码，交叉编译出 Windows、Linux、Android、ARM 等多个平台版本。

## 使用6818交叉编译

在我给你们的`Ubuntu16.4.0`已经配置好了基于6816 ARM架构开发平台交叉编译工具，所以后续需要运行在6818平台上的c语言代码都需要使用该工具进行交叉编译

```shell
arm-linux-gcc 源文件.c -o 目标文件

# 此时编译出来的目标文件就可以在6818开发板上运行
```
